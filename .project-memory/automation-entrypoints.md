# Automation Entrypoints

## 默认原则
高频请求先走稳定入口，不临场重造结构。入口只定义起步方式，不替代具体判断。

## 高频入口
- 阅读总结：先压缩主题、论点、关键概念，再整理为复习笔记或摘要。
- 论文 / 作业：先确认题目与评价标准，再给论点路径和提纲。
- 比较分析：先列比较维度，再用表格组织差异与判断。
- 汇报 / slides：先判断听众、时长和主信息，再设计讲述路径。
- 文本修改：先判断问题类型，再给修改版或改写建议。
- 文件交付：先确认主交付物，再走 `tool-delivery-routing.md`。
- 事实核对：先判断是否需要最新信息与来源，再决定是否查证。

## 默认收口
完成后给出一个最自然的下一步，例如扩写、文件化、做检查或生成示例。
