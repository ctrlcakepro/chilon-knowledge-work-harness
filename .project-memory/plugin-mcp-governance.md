# Plugin / MCP Governance

## 什么时候必须优先走专用能力
- 用户明确要求正式文件、表格、slides、PDF 审阅或其他特定交付物。
- 任务需要读取真实文件、检查仓库、验证页面或保留可下载成品。
- 用户要求修改外部系统中的现有内容。

## 什么时候先不要上重工具
- 用户只是在要思路、提纲、草稿或轻量判断。
- 任务可以用当前材料直接完成，且不会因为缺少工具导致交付物降级。
- 工具调用成本高于当前任务收益。

## 回退原则
- 首选能力不可用时，先查 `capability-fallback-matrix.md`。
- 回退时优先保持主交付物类型，例如 slides 任务优先给可迁移的 slides 结构，而不是改成普通说明。
- 明确说明哪些部分已完成，哪些部分因为能力限制只能保留为建议。
