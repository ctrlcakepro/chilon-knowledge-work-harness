# Rule Precedence｜规则优先级

Chilon 使用多层文件来组织 AI 行为，但这些文件不是编程语言里的继承系统。模型不会天然拥有严格的 `override` 语义，所以必须把冲突处理规则写清楚。

本文件定义 Chilon 的默认规则优先级，用来回答一个核心问题：当不同文件、不同层级或不同时间的指令互相冲突时，应该听谁的？

## 默认优先级

从高到低：

| 优先级 | 来源 | 说明 |
|---:|---|---|
| 1 | 当前用户明确要求 | 用户当前轮明确提出的目标、限制、语言、格式和交付物要求优先。 |
| 2 | 当前任务材料 | 用户刚提供的题目、原文、文件、作业要求、评分标准或上下文优先于历史偏好。 |
| 3 | 项目级 `AGENTS.md` | 定义当前项目最稳定的协作协议和行为边界。 |
| 4 | `PROJECT-WORKFLOW.md` | 定义常见任务的默认流程；如果和 `AGENTS.md` 冲突，以 `AGENTS.md` 为准。 |
| 5 | `.project-memory/` | 保存长期偏好、项目状态、决策记录和可复用资产；如果和当前要求冲突，以当前要求为准。 |
| 6 | `adapters/` 默认指令 | 平台适配包里的默认行为说明，只在没有项目级规则时作为入口。 |
| 7 | 通用模型习惯 | 模型自己的默认表达方式、常识性建议和风格偏好优先级最低。 |

## 核心原则

### 1. 当前明确要求高于历史记忆

如果用户当前说“这次只要中文”，即使历史偏好或某个 adapter 文件写过“默认中英双语”，也应以当前要求为准。

### 2. 具体任务材料高于通用 workflow

如果 `PROJECT-WORKFLOW.md` 说论文任务默认先给提纲，但用户提供的作业要求明确要求“一段完整短评”，则应直接按作业要求输出短评。

### 3. 项目级规则高于平台适配规则

`adapters/claude-desktop/`、未来的 `adapters/cherry-studio/` 只是迁移入口。真正安装或导入到某个项目后，项目自己的 `AGENTS.md`、Project instructions 或项目知识文件应当优先。

### 4. Memory 只保存稳定影响，不保存一次性命令

`.project-memory/` 不应该记录一次性聊天要求。如果某条 memory 和当前用户明确要求冲突，应当优先执行当前要求，并在必要时提示 memory 可能需要更新。

### 5. 冲突无法判断时，先说明冲突，再给最小选择

如果冲突会明显影响交付物方向，不要假装没有冲突。应简短说明冲突点，并给 2 到 3 个选择。例如：

```text
我看到两个可能冲突的要求：项目规则要求中文输出，但当前材料要求英文摘要。你想这次按哪一个？
A. 全中文
B. 全英文
C. 中文正文 + 英文摘要
```

## 常见冲突示例

| 冲突场景 | 默认处理 |
|---|---|
| 历史偏好要求中英双语，当前用户要求只中文 | 当前用户要求优先，只输出中文。 |
| `PROJECT-WORKFLOW.md` 要求先给提纲，用户要求直接给成稿 | 用户当前交付物要求优先，直接给成稿。 |
| `.project-memory/` 记录旧项目目标，当前任务明显进入新阶段 | 当前任务优先，并建议更新 `project-state.md`。 |
| Claude adapter 默认轻量输出，但项目 `AGENTS.md` 要求详细审稿 | 项目 `AGENTS.md` 优先。 |
| 通用写作习惯和作业评分标准冲突 | 作业评分标准优先。 |

## 文件维护建议

- 不要在多个文件里重复写同一条硬规则。
- 稳定跨任务规则放进 `AGENTS.md`。
- 常见任务流程放进 `PROJECT-WORKFLOW.md`。
- 长期偏好和项目状态放进 `.project-memory/`。
- 平台迁移说明放进 `adapters/`。
- 如果某条规则只服务一次任务，不要写入项目规则层。

## 最小心智模型

记住一句话即可：

> 当前任务优先于历史记忆，项目规则优先于平台默认，具体材料优先于通用流程。
