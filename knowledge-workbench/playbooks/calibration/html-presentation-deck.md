# Case：HTML 组会演示 / 风格化 deck

## 任务信号
- 用户要 HTML 格式 PPT / deck / slides
- 有风格词，比如“BMW 性冷淡风”“像 Claude 一样高级”
- 明确是组会、展示、明天要讲
- 用户在意全屏、提词器、讲者视图、审美和排版

## 上次容易做偏的地方
- 把演示任务做成普通网页
- 没用现成 `html-ppt` 能力，手搓弱化版页面
- 只有观众页面，没有 presenter mode / notes / 提词支持
- 审美翻译太表面，做出“土味 + AI 模板味”
- 信息平均铺开，像产品首页而不像演示

## 正确路径
- 先识别为“演讲交付任务”，不是普通设计页
- 直接走 `html-ppt` 及其支持讲者模式的模板与运行时
- 先做讲述路径，再做页面，再补 notes / presenter mode
- 抽象风格词先过审美翻译器，再落到版式和禁忌项
- 交付前检查全屏、翻页、notes、首屏气质和排版节奏

## 永久规则
- HTML 演示默认优先 `html-ppt`，不要手搓普通网页冒充幻灯片。
- 用户一旦提到演讲、明天要讲、提词器、speaker notes，这些都算核心功能要求。
- 抽象风格词先翻译，不直接赌视觉。

## 开始前预警问题
- 我是不是又把演示做成网页了
- 我是不是遗漏了全屏、翻页、讲者视图
- 我是不是把“高级”误做成了炫技皮肤
- 首页看上去像演示封面，还是像官网 banner
