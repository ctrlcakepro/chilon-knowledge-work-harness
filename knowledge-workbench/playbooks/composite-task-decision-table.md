# 复合任务优先决策表

目标：当一个请求同时包含两种以上任务意图时，快速判断谁是主任务、谁是次任务，避免把复合任务做成平均发力、主次不清的结果。

## 基本原则
- 复合任务默认只选 1 个主任务。
- 其余任务作为次任务、附加验收项或交付形态约束。
- 主任务决定：
  - 主工作流
  - 结果的第一判断
- 次任务决定：
  - 补充功能检查
  - 输出格式或呈现方式

评分卡不在这里展开维护。
复合任务一旦裁决完成，默认转到 `knowledge-workbench/playbooks/scoring-routing.md`。

## 默认裁决顺序
1. 先看用户真正要解决什么问题。
2. 再看最后要交付什么成品。
3. 再看什么缺口最会导致“不能交”。
4. 最后才看审美、风格、语气等增强项。

一句话判断：
- 哪一部分失败，会最直接让这次任务失效，那一部分通常就是主任务。

## 主次裁决硬规则
### 1. 文件交付不是默认主任务
如果用户要求 PDF、Word、PPT、HTML、表格等文件：
- 只有当用户最在意“文件真实生成、打开可用、格式正确”时，`执行与文件交付` 才升为主任务
- 否则文件交付默认是次任务或附加验收项

### 2. 演示任务优先保“能讲”
如果任务同时包含内容整理、审美优化和演示交付：
- `演示与讲述交付` 默认优先于 `设计与呈现`
- `设计与呈现` 默认作为次任务
- 如果缺少全屏、翻页、notes、presenter mode，则先修演示交付，不先修美观

### 3. 查询任务优先保“核对”还是“判断”
如果任务同时包含查资料与给判断：
- 如果用户核心诉求是“真不真、有没有出处、现在到底怎样”，主任务是 `查询与核对`
- 如果用户核心诉求是“基于信息帮我选、帮我判断怎么做”，主任务是 `比较与判断`

### 4. 写作任务优先保“成稿”
如果任务同时包含读材料、整理信息、扩写正文：
- 只要最终目标是交一篇能用的文章，主任务默认是 `写作与成稿`
- `理解与整理` 作为次任务，为写作提供支撑

### 5. 评估任务优先保“结论”
如果任务同时包含阅读材料、指出问题、生成正式意见：
- 主任务默认是 `评估与诊断`
- 文件生成通常是次任务
- 不要因为要交 PDF / Word，就把评估任务降级成排版任务

### 6. 计划任务优先保“可执行”
如果任务同时包含分析现状、比较方案、给行动路径：
- 如果最终目标是“接下来怎么做”，主任务默认是 `方案与计划`
- `比较与判断` 作为次任务，为计划服务

## 高频复合任务决策表
### 1. 读材料 + 写文章
- 主任务：`写作与成稿`
- 次任务：`理解与整理`
- 先保：文章能成型、结构完整、可直接使用

### 2. 查资料 + 给判断
- 主任务：`查询与核对` 或 `比较与判断`
- 次任务：另一个作为补充
- 裁决规则：
  - 重点在真伪与出处：`查询与核对` 为主
  - 重点在选择与结论：`比较与判断` 为主

### 3. 评估材料 + 生成正式意见
- 主任务：`评估与诊断`
- 次任务：`执行与文件交付`
- 先保：结论明确、问题排序、建议可执行
- 后保：文档版式、导出格式

### 4. 做演示稿 + 做好看
- 主任务：`演示与讲述交付`
- 次任务：`设计与呈现`
- 先保：能全屏、能翻页、能讲、主线清楚
- 后保：气质、留白、排版高级感

### 5. 整理信息 + 给执行方案
- 主任务：`方案与计划`
- 次任务：`理解与整理`
- 先保：步骤、优先级、依赖、可执行性
- 后保：背景整理是否更完整

### 6. 查询事实 + 生成成稿
- 主任务：`写作与成稿`
- 次任务：`查询与核对`
- 先保：文本成型
- 同时强制保：事实不乱写、出处不乱判

### 7. 审美优化 + 页面实测
- 主任务：`执行与文件交付` 或 `设计与呈现`
- 裁决规则：
  - 如果用户要的是“页面真实可用、交互真实成立”，`执行与文件交付` 为主
  - 如果用户要的是“视觉方向、版式气质、怎么改更高级”，`设计与呈现` 为主

### 8. 比较方案 + 给推荐路线
- 主任务：`比较与判断` 或 `方案与计划`
- 裁决规则：
  - 如果重点是“哪种更好”，`比较与判断` 为主
  - 如果重点是“接下来怎么落地”，`方案与计划` 为主

## 冲突裁决表
### 1. 内容质量 vs 文件格式
- 先保内容质量
- 除非用户明确说“我要的是成品文件本身”

### 2. 演示功能 vs 审美气质
- 先保演示功能
- 再保审美气质

### 3. 查询准确性 vs 输出流畅度
- 先保查询准确性
- 再保表达顺滑

### 4. 评估结论 vs 文档排版
- 先保评估结论
- 再保文档排版

### 5. 方案可执行性 vs 背景铺陈完整度
- 先保方案可执行性
- 再补背景铺陈

## 与其他层的接线
### 1. 与工具路由
工具路由前先识别主任务，工具路由后再补次任务要求。

### 2. 与评分路由
主次任务裁决完成后，再到 `scoring-routing.md` 选择主评分卡与次评分卡。

### 3. 与评分输出
评分结果优先围绕主任务报告，不要让次任务评分喧宾夺主。

## 最小执行口径
遇到复合任务时，默认先回答这 5 个问题：

- 这次真正的主任务是什么
- 什么只是交付形式，不是任务本体
- 什么只是增强项，不是主判据
- 哪个失败最会导致不能交
- 裁决完后是否需要转到 `scoring-routing.md`
