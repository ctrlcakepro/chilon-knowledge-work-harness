# Benchmark 任务集

目标：提供一组稳定、跨任务族的校准题，用来测试 harness 在真实知识工作中的表现。

## 使用说明
- 每次不必全跑，默认选 4 到 8 个任务即可
- 如果刚改了某一层，优先跑跟那一层最相关的任务
- 每个任务都记录：主任务、主评分卡、是否过线、失败落点

## Task 1：阅读总结
- `任务类型`：理解与整理
- `样例请求`：根据这篇长文/章节材料，帮我整理主题、核心论点、论证结构和可继续展开的角度。
- `预期主任务`：阅读总结
- `预期主评分卡`：`理解与整理`
- `通过信号`：
  - 先抓主旨，不只是复述细节
  - 结构清楚
  - 能继续拿去写作或复习

## Task 2：比较判断
- `任务类型`：比较与判断
- `样例请求`：比较 A 与 B 两条方案/理论/工具路线，并直接告诉我更推荐哪一个以及为什么。
- `预期主任务`：比较与判断
- `预期主评分卡`：`比较与判断`
- `通过信号`：
  - 比较维度先行
  - 结论明确
  - 不只是列优缺点，而是真有判断

## Task 3：查询核对
- `任务类型`：查询与核对
- `样例请求`：帮我查清这条事实现在到底是否成立，并区分已确认信息和不确定信息。
- `预期主任务`：查询与核对
- `预期主评分卡`：`查询与核对`
- `通过信号`：
  - 明确区分事实、推测、待核对
  - 不把不确定写成既定事实
  - 结论不含混

## Task 4：写作成稿
- `任务类型`：写作与成稿
- `样例请求`：根据这些材料，写一篇可直接使用的课程作业/短报告/申请文本。
- `预期主任务`：写作与成稿
- `预期主评分卡`：`写作与成稿`
- `常见次评分卡`：`理解与整理`
- `通过信号`：
  - 结构完整
  - 像成稿，不像提纲
  - 不因为材料不完整就退化成大量说明

## Task 5：评估诊断
- `任务类型`：评估与诊断
- `样例请求`：看这份材料/方案/论文，判断它是否达标，并按正式评估口径给修改建议。
- `预期主任务`：评估与诊断
- `预期主评分卡`：`评估与诊断`
- `通过信号`：
  - 先给结论
  - 问题有排序
  - 建议可执行

## Task 6：演示交付
- `任务类型`：演示与讲述交付
- `样例请求`：根据这份材料做一个可讲的 PPT / HTML deck，要求全屏、翻页、notes 或 presenter 支持。
- `预期主任务`：演示与讲述交付
- `预期主评分卡`：`演示与讲述交付`
- `常见次评分卡`：`设计与呈现`、`执行与文件交付`
- `通过信号`：
  - 没把演示做成普通网页
  - 主线清楚
  - 讲者需求进入定义

## Task 7：审美翻译
- `任务类型`：设计与呈现
- `样例请求`：我要“像 Claude 一样高级”/“BMW 性冷淡风”/“学术但不土”，先给我方向判断再展开方案。
- `预期主任务`：设计与呈现
- `预期主评分卡`：`设计与呈现`
- `通过信号`：
  - 先做风格翻译，不直接套皮肤
  - 能说清主视觉、层级、禁忌项
  - 不出现明显 AI 模板味

## Task 8：文件交付
- `任务类型`：执行与文件交付
- `样例请求`：把这份内容输出成 PDF / Word / 表格 / PPT，并确保打开可用、格式正确。
- `预期主任务`：执行与文件交付
- `预期主评分卡`：`执行与文件交付`
- `通过信号`：
  - 真正生成文件
  - 文件可用
  - 没把文件任务降级成口头说明

## Task 9：复合任务
- `任务类型`：复合任务裁决
- `样例请求`：查一下资料后帮我判断哪条路更适合，并整理成一页执行方案。
- `预期主任务`：`比较与判断` 或 `方案与计划`
- `预期主评分卡`：按主任务确定
- `常见次评分卡`：`查询与核对`、`理解与整理`
- `通过信号`：
  - 没有平均展开
  - 主任务裁决清楚
  - 次任务只做支撑，不抢中心

## Task 10：失败闭环
- `任务类型`：失败复盘与治理
- `样例请求`：这次结果明显做偏了，请判断偏差属于哪一层，并把经验沉淀成后续规则。
- `预期主任务`：失败闭环
- `预期主评分卡`：通常是 `评估与诊断`
- `通过信号`：
  - 先承认偏差
  - 能指出正确路径
  - 能说明该补到哪一层规则
