# 全局知识工作台

目标：把高频知识工作任务沉淀成可复用的全局骨架，减少每次临场重组。

适用范围：
- 阅读与总结
- 论文与作业写作
- 审稿与规范评估
- 汇报与展示准备
- 设计方向判断
- 选项式追问
- 成品交付前自检
- 成品交付前验收评分

目录说明：
- `architecture/`：harness 分层、评分与优先级
- `architecture/health-check.md`：当前系统健康状态与控长重点
- `questioning/`：选项式询问模板
- `benchmarks/`：小型 benchmark 任务集、选题映射与结果模板
- `benchmarks/selection-guide.md`：体检或全局改造后该优先跑哪些 benchmark task
- `templates/`：高频任务骨架
- `style/`：审美方向库
- `playbooks/`：路由、复合任务裁决、评分映射、评分输出、失败闭环、规则治理、续接与质量闸门
- `playbooks/index.md`：playbook 轻量导航入口
- `playbooks/health-check-trigger.md`：周期性体检触发条件与默认动作
- `playbooks/health-check-report-template.md`：体检结果的统一输出模板

使用原则：
- 先按 `architecture/` 明确当前任务所在层级与执行顺序，再调用具体模板。
- 如果任务本身是复合任务，先裁决主任务与次任务，再进入具体模板。
- 先用模板稳住任务骨架，再根据具体材料调内容。
- 工具路由确定后，默认继续选出主评分卡与必要的次评分卡。
- 中等及以上复杂任务，默认在质控后再过一遍评分层。
- 需要显式给出验收判断时，默认使用统一的评分输出格式。
- 如果任务已经做偏，默认用失败闭环模板判断这次经验该沉淀到哪一层规则。
- 如果要继续沉淀系统规则，默认先过一遍规则治理协议，避免全局规则越积越胖。
- 如果最近持续在加规则或某个中层文件明显变胖，默认先过一遍体检触发器，再决定要不要继续扩展。
- 如果已经触发体检，默认按统一体检模板输出结果，避免每次说法不一。
- 如果做了较大结构修改，或中等以上体检后改了行为入口，默认再按 `benchmarks/selection-guide.md` 跑一轮 benchmark，验证系统是在变强而不是变复杂。
- 模板是工作台，不是僵硬套版。
- 如果用户已有明确格式，优先贴近用户目标，而不是强套默认结构。
